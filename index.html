JobRadar Blueprint & System Report

Complete guide to production scaling, security policies, and custom prompts.

1. Scraping Strategy & Zero-Cost Architecture

To scale this system without recurring API fees, we implement a self-hosted pipeline using node scripts and free cloud databases:

  • Local CLI Runner (cheerio/axios): Instead of browser engines which require significant memory overhead, the RSS feeds of major boards (LinkedIn, WeWorkRemotely, Remote.co) are parsed in node using XML readers. This requires 0 proxies or paid subscription modules.
  • Cloudflare & Anti-Bot Bypass: For strict portals like Naukri, the script uses customized Request Headers (rotating User-Agents, mimicking browser sec-ch-ua signatures). Additionally, a cookie parser pulls active sessions directly from local browsers to bypass session tokens.
  • Free Hosting & Database: The frontend code is compiled as static HTML/CSS/JS and can be hosted completely free on GitHub Pages, Vercel, or Netlify. Storing jobs permanently can be achieved using the free-tier PostgreSQL database on Supabase or Neon (up to 3GB/0.5GB data, which supports over 100,000 job records).

2. Database & Semantic Vector Search Indexing

For high-performance filtering and semantic matching in production, we recommend the following free schema:

  • PostgreSQL Database: Store listings using standard fields (Title, Company, Location, Description, Apply URL, Salary, AmbitionBox Score).
  • pgvector for Semantic Resume Matches: Instead of simple substring matching, you can generate vector embeddings of job descriptions and user resumes using free-tier Gemini API embeddings (text-embedding-004). pgvector performs cosine similarity queries directly inside PostgreSQL, matching candidates with roles that don't share exact keywords but have matching conceptual meaning.

3. Security Blueprint & Client Protection

Maintaining security on a free hosting tier is vital to prevent credential theft or browser hijacking:

  • Environment Variable Isolation: In a production deploy, scraping scripts and AI endpoints should be housed in a small backend (Node.js/Express) running on Render Free Tier or Railway Developer Plan. Sensitive keys (like Google Gemini API keys) are kept in .env variables on the server. The frontend client calls the backend gateway, securing the keys from being inspected in the browser.
  • Cross-Site Scripting (XSS) Mitigation: All aggregated jobs descriptions contain raw HTML tags from web scrapers. We sanitise this input before rendering to prevent malicious scripts from executing. In this prototype, we use safe text nodes or custom tag replacers (e.g. escapeHTML()).
  • Rate-Limiting (DDoS Protection): Free backend endpoints should use libraries like express-rate-limit to limit client requests to 15 per minute, matching the free tier of the Gemini API and protecting the server from overloading.

4. LLM Prompt: System Rebuilder

Use the following highly structured system prompt in Gemini or ChatGPT to recreate or scale this entire application with high visual and layout fidelity:

You are an expert Frontend Architect specializing in glassmorphism and dashboard design systems. Create a single-page Job Search Aggregator application using Vanilla HTML, CSS, and JS.

1. Palette: Slate dark backgrounds (#090d16), glassmorphic card overlays (rgba(30, 41, 59, 0.45)) with custom back-drop blurs. Glowing borders utilizing gradients of violet (#8b5cf6) to cyan (#06b6d4).
2. Navigation: Sticky sidebar displaying tabs for: Job Dashboard, ATS Matcher, Kanban tracker, and Scrapers console.
3. Fluid UX: Smooth 60FPS CSS transitions, custom scrollbars, glowing keyframe animations, and linear shimmer skeletons.

LOGICAL REQUIREMENTS:
